# 06. 디자인 시스템

Phaeton ERP 대시보드 디자인 규칙. 모든 UI 작업 전 이 문서를 참조할 것.

---

## 1. 색상 체계

색상은 **의미 기반**으로만 사용한다. 장식용 색상은 금지.

### 시맨틱 색상

| 의미          | 색상 토큰   | 용도                                 |
| ------------- | ----------- | ------------------------------------ |
| 정상 / 완료   | `green-500` | 완료된 마일스톤, 정상 상태 표시      |
| 진행중 / 정보 | `blue-500`  | 현재 활성 작업, 정보성 알림          |
| 경고 / 주의   | `amber-500` | 기한 임박, 주의 필요 항목            |
| 위험 / 지연   | `red-500`   | 지연, 차단, 오류 상태                |
| 비활성 / 대기 | `gray-300`  | 아직 시작하지 않은 항목, 비활성 요소 |

### 프로젝트 유형 색상

| 유형       | 색상 계열 | 예시                                 |
| ---------- | --------- | ------------------------------------ |
| 태양광     | `amber`   | `bg-amber-100`, `text-amber-700`     |
| 풍력       | `sky`     | `bg-sky-100`, `text-sky-700`         |
| ESS        | `emerald` | `bg-emerald-100`, `text-emerald-700` |
| 하이브리드 | `violet`  | `bg-violet-100`, `text-violet-700`   |

> 위 색상 외의 색상을 UI에 사용하지 않는다. 배지, 차트, 상태 표시 등 모든 곳에 동일 규칙 적용.

---

## 2. 상태 라벨

상태 텍스트는 **한국어 고정**. 영문 라벨을 사용자에게 노출하지 않는다.

### 마일스톤 상태

| 코드      | 라벨   | 색상        |
| --------- | ------ | ----------- |
| `done`    | 완료   | `green-500` |
| `active`  | 진행중 | `blue-500`  |
| `pending` | 대기   | `gray-300`  |
| `blocked` | 차단   | `red-500`   |
| `skipped` | 건너뜀 | `gray-300`  |

### 프로젝트 상태

| 코드           | 라벨   |
| -------------- | ------ |
| `planning`     | 기획   |
| `permit`       | 인허가 |
| `construction` | 시공   |
| `testing`      | 시운전 |
| `cod`          | 운영   |

---

## 3. 타이포그래피

- **최소 글씨 크기: 12px** (`text-xs` 이상). 12px 미만 사용 금지.
- 기본 본문: 14px (`text-sm`)
- 제목/헤더: 필요에 따라 `text-base` ~ `text-xl` 사용
- 폰트: 시스템 기본 폰트 스택 (별도 웹폰트 없음)

---

## 4. UI 원칙

### 기본 방침

- **다크 모드 없음** -- 라이트 모드 단일 지원
- **그라디언트, 그림자, 애니메이션 금지** -- 대시보드에 장식적 효과를 넣지 않는다
- 실용적이고 정보 밀도가 높은 대시보드를 지향

### 컴포넌트 선택 순서

1. **shadcn/ui (Radix) 컴포넌트**를 우선 사용
2. shadcn/ui에 없는 것만 **Tailwind CSS**로 직접 구현
3. 별도 UI 라이브러리 추가 금지

### 레이아웃

- 대시보드는 카드 기반 그리드 레이아웃
- 반응형은 Tailwind 브레이크포인트 (`sm`, `md`, `lg`, `xl`) 활용
- 여백/간격은 Tailwind 유틸리티 클래스로 통일

---

## 5. 컴포넌트 가이드

자주 사용하는 shadcn/ui 컴포넌트와 용도.

| 컴포넌트                             | 용도                                |
| ------------------------------------ | ----------------------------------- |
| `Card`                               | 대시보드 위젯 래퍼                  |
| `Table` / `DataTable`                | 프로젝트 목록, 마일스톤 목록        |
| `Progress`                           | 공정률, 진행 상태 표시              |
| `Badge`                              | 상태 라벨 (시맨틱 색상과 함께 사용) |
| `Input`, `Select`, `Textarea`        | 폼 입력 요소                        |
| `Tabs`                               | 페이지 내 섹션 전환                 |
| `Sheet`                              | 항목 상세 슬라이드 패널             |
| `Dialog`                             | 모달 다이얼로그                     |
| `Skeleton`                           | 로딩 상태 표시                      |

### Badge 색상 매핑 예시

```tsx
const statusColor: Record<string, string> = {
  done: 'green',
  active: 'blue',
  pending: 'gray',
  blocked: 'red',
  skipped: 'gray',
}
```

### 프로젝트 유형 Badge 예시

```tsx
const typeColor: Record<string, string> = {
  solar: 'amber',
  wind: 'sky',
  ess: 'emerald',
  hybrid: 'violet',
}
```
